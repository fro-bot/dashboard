# Product

**Fro Bot Dashboard** — a read-only monitoring and operator-control surface for Fro Bot, the autonomous GitHub agent.

## Lane

Product (an authenticated operator tool), not a marketing surface. The audience is a single operator (Marcus) watching Fro Bot's cross-repo footprint and, behind a flag, driving the operator-control loop (launch / observe / approve).

## Audience

One technical operator. No onboarding funnel, no SEO, no public landing — the user is already authenticated and knows the domain. Optimize for at-a-glance status comprehension, fast triage, and trustworthy security affordances over discoverability or persuasion.

## Voice

Terse, operational, precise. Status over prose. Never marketing-speak. Surface what is real (and what is stale, failed, or redacted) honestly; never imply availability that isn't there.

## Brand

Afrofuturism × cyberpunk — organic soul meets machine precision. Deep-purple/void dark-default, with cyan (action), magenta (emphasis), and amber (highlight) accents, plus deliberate glow on focal interactive elements. The full system is `assets/styleguide.md`; the executable tokens are `web/src/styles/tokens.css`. These brand elements — deep-purple backgrounds, cyan/magenta accents, focal glow — are intentional identity, not decoration to be flattened.

## Anti-references

- Generic SaaS-dashboard blandness (gray-on-gray, undifferentiated cards).
- AI-slop tells used *without* brand intent: gratuitous purple gradients, indiscriminate dark glows on every element, bounce easing everywhere. The Fro Bot aesthetic uses purple and glow *deliberately and sparingly* (glow on focal elements only) — that is the line between brand and slop.
- Marketing gloss, hero-pitch copy, persuasion patterns. This is a tool.
